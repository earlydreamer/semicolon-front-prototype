# Pull Request: 프론트엔드-백엔드 서비스 통합 (#23)

## 📌 개요
본 PR은 프론트엔드 목업 상태에서 실제 백엔드 API와의 연동을 완료하고, 인증 영속성 및 데이터 필드 정합성을 확보하기 위한 작업 내용을 포함합니다.

## 🚀 주요 변경 사항

### 1. 인증 시스템 고도화 및 영속성 처리
- **인증 영속성 유지**: `Zustand`의 `persist` 미들웨어를 활용하여 새로고침 시에도 로그인 상태가 유지되도록 개선.
- **초기화 로직 추가**: `App.tsx`에서 `isInitialized` 플래그를 도입하여 토큰 검증(`getMe()`) 완료 전까지 UI 렌더링을 제어, 깜빡임 및 권한 오류 방지.
- **자동 로그아웃**: `api.ts`에 401 Response Interceptor를 추가하여 토큰 만료 시 전역 상태 초기화 및 로그인 페이지 이동 로직 구현.
- **관리자 권한 강화**: `isAdminAuthenticated` 유도 상태를 추가하여 관리자 전용 기능 접근 제어 강화.

### 2. 백엔드 DTO 필드명 전수 연동 (Data Parity)
- **상품(Product)**: `thumbnailUrl`, `productUuid`, `imageUrls` 등 백엔드 규격에 맞춰 타입 및 매핑 로직 수정.
- **주문(Order)**: `OrderResponse` 규격에 맞춰 주문 내역, 상세 보기 데이터 필드 정합성 확보.
- **사용자(User)**: 프로필 업데이트 시 `name` 필드 사용 및 `userUuid` 연동.

### 3. 서비스 레이어 확장
- **orderService**: 배송 정보 수정, 송장 번호 업데이트, 주문 상태 변경 기능을 `OrderController` 규격에 맞춰 구현.
- **userService**: 비밀번호 변경(`updatePassword`) 및 상세 프로필 조회 연동.
- **authService**: 기존 `authStore` 로직을 별도 서비스로 추출하여 모듈화.

### 4. UI/UX 개선
- **비밀번호 변경 모달**: 마이페이지 내 실 서비스 연동 완료.
- **주문 내역 카드**: 목업 데이터와 실제 API 데이터를 동시에 지원하도록 정규화 로직 적용.
